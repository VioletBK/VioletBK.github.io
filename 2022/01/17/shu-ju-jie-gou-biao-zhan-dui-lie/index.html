线性表

考纲要求

线性表的概念

线性表：零个或多个数据元素的有限序列

线性表根据在计算机的储存方式可以分为两种：

  • 顺序线性表
  • 链式线性表

插入算法的实现

基本思想如下：

  • 插入位置不合理报错，如插入a[0]前或者a[length+1]; if(i<0 || i > length + 1)
  • 如果插入后，线性表的长度大于最大长度，报错或者动态增加容量
  • 插入从后向前，从a[length-1]到a[i-1]依次后移一位。
  • 将元素插入。
  • length+1

删除算法的实现

基本思想如下：

  • 删除位置不合理，就是说不在0到length-1的范围内。
  • 取出删除元素
  • 从被删除i位置，a[i]依次向后向前移一位
  • 长度减一

插入与删除的时间复杂度

最好情况，插入删除都在顺序表尾部。

此时时间复杂度为O(1)，总体平均为O(n-1/2);所以时间复杂度为O(n)

顺序线性表的优缺点


顺序线性表的优缺点：
优点：
- 可以快速获取下标的数据元素，时间复杂度为O(1)
- 逻辑关系是一对一的关系，连续存储单元足以储存，不需要增加额外的存储空间

缺点：
- 插入和删除操作需要移动大量的元素，时间复杂度为O(n)
- 线性表的存储空间大小难以确定，并且不好扩展
- 造成存储空间碎片

单链表

单链表的读取（感觉有点问题）

单链表不像顺序表一样通过数组构建，因此无法直接查找特定位置的数据元素。

  • 声明一个节点p执行链表的第一个节点。（问题就来了，第一个节点是否代表头节点，但是头节点一般是不存储数据的，所以我们可以假设第一个节点不为头节点，应为p=L->next）
  • 声明一个j，初始化设置为1(0)，设置循环条件(p!=null && j < i)。（感觉这里且或者或都可以，还是为了保障安全性，还是使用或）。
  • 随着循环的进行j++，p= p->next
  • 如果到了i位置，p为Null说明到了链表的末尾，即没有查询到
  • 否则查询到

插入

e插入p节点之后。

核心代码为


e->next = p->next;
p->next = e;

基本思想如下

  • 插入位置是否合理
  • 遍历到要插入的位置（遍历使用上面的取节点），否则报错
  • 到达位置之后，创建一个新的节点，并将数据传给它。
  • 使用核心代码进行插入。

删除

核心代码为


t = p->next;
p->next = p->next->next;
delete(t);

单链表的整体创建

首插法：即将新建的节点插入到头指针与上一个新节点之间


LinkedList l = new LinkedList(ElementType data);
l->next = List -> next;
List -> next = l;

尾插法：更符合我们的逻辑，即将新建节点插入到链表的最后。

其实这里逻辑上符合常规，但是实际算法比较麻烦。因为我们需要不断指向尾部，如果使用遍历的方式的话，必然会造成很高的时间复杂度，所以我们可以创建一个尾节点不断指向链表的尾部。


void createNewLinkedList(LinkedList *L, int n){
  LinkedList * r = *L;    // 指向链表尾部，链表一开始只有一个头节点即*L
  for(int i = 0; i < n; i++){
    LinkedList *p = new LinkedList(1);
    r->next = p;    
    r = p;
  }

}

单链表的整表删除

删除的思路在于：

  • 声明节点p和q
  • 将第一个节点赋值给p
  • 将p->next赋值给q，然后释放p，再将q赋值给q

循环链表

基本概念

将单链表中终端节点的指针端有空指针改为指向头节点，就使整个单链表形成一个环，这种头尾相接的单链表成为单循环链表，简称循环链表（circular linked list）

循环链表和单链表的主要差异就在于循环的判断条件上，现在为p->next 是否等于头节点。

尾指针

我们可以设置一个尾指针指向终端节点，此时查找开始节点和终端节点都十分的方便。终端节点表示为rear，开始节点表示为rear->next

合并两个链表，当有尾指针后，合并两个链表会十分方便。

原本为

合并操作如下，*p = rearA->next; rearA->next = rearB->next->next,这里是因为要指向第一个节点，而非头节点; rearB->next=p ;

双向链表

概念

双向链表（double linked list）实在单链表的每个节点中，在设置一个指向其前驱节点的指针域。所以双向链表有两个指针域，一个指向前驱，一个指向直接后继。

结构如下。

插入

因为有两个指针域，所以它要修改两个指针变量。插入S，找到p。


s->next = p->next; 
s->prior = p; 
s->next->prior = s(或者p->next->prior=s);
p->next = s;

删除

删除操作可以分解为，前一个节点的后驱变为后一个节点即s->prior->next=s->next; 后一个节点的前驱变为前一个节点，s->next->prior = s->prior;然后释放s,delete(s);

栈与队列

栈的概念

很多软件中可以实现撤销(undo)操作，这里就是使用栈。后进先出。

栈(stack)是限定金在标为进行插入和删除操作的线性表

我们把允许插入和删除的一端称为栈顶(top)，另一端称为栈底(bottom)，不含任何数据元素的栈称为空栈。栈又被称为后进先出(Last In First Out)的线性，简称为LIFO结构。

进栈与出栈

1、2、3依次入栈，会有哪些出栈顺序呢？

321、123、213、132、231

从这里我们可以得到，312必不可能，因为3第一个出栈代表全部数都已经进栈，然后其余数字再依次出栈，所以2必然会在1之前。那么我们对此进行规律总结。如1、2、3、4、5中，4先出栈，1到3必然按照从大到小的顺序进行出栈。同样的规律用于子序列中。

栈的顺序存储结构以及实现

栈的顺序存储结构

存在栈的最大长度StackSize。栈顶位置top必须小于StackSize。

当栈存在一个元素是，top等于0，因此通常把空栈的判断条件定位top等于-1

栈的结构定义


struct {
  elementType data[MAXSIZE];
  int top;        // 用于栈顶指针
}

进栈操作

原理很简单

S->top++;
S->data[top] = e;

出栈操作

与进栈操作类似

*e = S->data[S->top];
S->top–;

栈的链式存储结构及实现

栈的链式存储结构

栈只用栈顶来做插入和删除操作，那么问题就来了，宅顶放在链表的头部还是尾部。由于单链表有头指针，而栈顶指针也是必须的，那么干脆合二为一。

同时由于已经有栈顶在头部了，相应的头节点也就失去了意义。

链栈结构代码


struct StackNode{
  elementType data;
  struct StackNode * next;
}

struct LinkedStack{
  LinkedStackPtr top;
  int count;
}

链栈进栈操作

使用首插发。


Push(LinkedStack *S, elementType e){
  StackNode * s = new StackNode;
  s->data = e;
  s->next = S->top;
  S->top = s;
  S->count++;
}

链栈的出栈操作


Pop(LinkedStack *S, elementType *e){
  StackNode *p = S->top;
  S->top = S->top->next;
  e = p->data;
  free(p);
  S->count--;
}

int Fbi(int i){

    return i:0?0:1;
  }

栈的应用——四则运算

直接进行四则运算太过于复杂，所以这里使用后缀表达式，计算后缀表达式。

后缀表达式计算规则

运算规则：从左到右遍历，遇到数字压栈，遇到符号就将栈顶两个数字出栈，进行运算，结果压栈，然后一直得到结果。

如：9 3 1 -3 * + 10 2 / +

  1. 9 3 1依次进栈
    • ，1，3出栈 3-1=2，2进栈
  3. 3进栈
  4. ，2与3进栈， 23 = 6进栈
  5. +,9和6出栈，9+6 = 15进栈
  6. 10 ，2 进栈
  7. / 2，10 出栈 10/2=5进栈
  8. +，5，15出栈 5+15=20
