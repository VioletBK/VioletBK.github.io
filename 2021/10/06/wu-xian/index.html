IEE 802.11

定义了WLAN Mac和物理层协议

目标

  • 与有线服务交互

  • 更高的传输速率

  • 更高可靠性

  • 持续的网络连接

设立的协议

多个物理层协议：IR，FHSS，DSSS，OFDM，FDMA

MAC 子协议：

MAC管理协议和服务

结构

它以蜂窝网络（cellular architecture）为基础，划分为多个BSS（ Basic Service Set），每个BSS由AP控制（Access Point）

WLAN可以由一个BSS和一个AP构成。但是，一般来说它是由多个BSS组成，其中AP是有名叫分布式系统主干（DS）连接的，例如因特网。

相互连接的WLAN由BSSs，AP，DS组成构成802网络，又被称为ESS（Extended Service Set）

IBSS不需要AP，可以直接进行互相交流

连接模式

Infrastructure mode

端与端之间交流通过一个，或者多个通过有线进行连接的AP进行连接。

  • 端选择一个AP并与其连接

  • AP支持漫游（AP support roaming）

  • AP提供功能

    时间同步：指引

    能源管理：

    点协调

Ad hoc mode

多跳，不需要基础设备，不需要AP，端与断就可以互相连接。

IEEE标准的数字体系

802.* 与 802.1*适用与所有的802技术。

  • 802.11*是适用于所有WiFI设备，但是不适用于基于802.15的ZigBee设备

  • 标准中都是大写字母的是基础标准。

    • 比如IEEE 802.1AB-2009

  • 标准中有小写的字母的是拓展/添加/修改的。这些修改过的协议一般是由多个基础协议汇总的（Merge）

    • 比如 IEEE 802.1w - 2001是由IEE 802.1D-2004汇总

  • 标准是由数字字母排序的

    • IEEE 802.1a … IEEE 802.1z，802.1aa …. 802.1ab

ISM

WLAN 一般使用 ISM 波段

2.4GHZ

IEEE 802.11 b/g/n 定义了2.4GHz频段的WLANs（2.4 - 2.4835GHz, 83.5MHZ）

2.4GHz频段被分了14个载波通道（carrier channel），它们之间部分重叠。每个中心频道的间隔是5Mhz

在802.11b中，每个通道占据了22MHz的频谱范围

在802.11g/n中，每个通道占据了20Mhz的频道范围

5GHZ

IEEE 802.11 a/n/ac 定义了5GHz频段

2002年有5个不重叠的通道在5.8GHz的频段工作（5.725 -5.875GHz，150MHz）

2012年在5.15GHz到5.35GHz 添加了8个通道。但是，他们仅仅在室内使用。每个通道占据20Mhz。

6GHz

6GHz为wifi和5G带来了新的带宽。

目前来说，6GHZ在大部分的国家和地区都处于研究阶段。仅仅有少部分设备支持6G。

60GHZ

60GHz占据了9Ghz的频段（57-66Ghz），同时它是毫米波。

用于者四个频段，每个占据约2Ghz。

相对于其他频段，60GHz有着如下优点。

  1. 大的频谱：9GHz

    • 2.4GHz是80MHz，5GHz是350Mhz。
    • 用相同的方式调制和编码模式，更易实现千兆比特数据传输（gigdbit data rate）
    • 复杂的256-QAM不再需要，QPSK或者16-QAM即可

  2. 更容易实现波束成形，芯片上的天线阵列。

  3. 更小的干扰，不需要穿墙。

  4. 指向天线（directional Antennas），空间复用更容易

  5. 更强的传输能量

同样的它也有缺点

  1. 更大的天线：

    • 更强的氧气吸收
    • 需要更强的能量
    • 需要更高的天线获取率，指向性天线（容易被遮挡）
    • 更短的距离

  2. directional deafness：（不知道咋翻译好）

    • 载波探测不可能
    • RTS，CTS不在工作
    • Multicast困难

  3. 容易被遮挡

802.11a 和 802.11b

802.11 b

使用了DSSS（Direct Sequence Spread Spectrum）。2.4Ghz

数据速率为5.5Mbps和11Mbps

使用了CCK和PBDC调制机制。

它使WIFI真正流行。
