流程控制函数

流程处理函数可以根据不同的条件，执行不同的处理流程，可以在SQL语句中实现不同的条件选择。 MySQL中的流程处理函数主要包括IF()、IFNULL()和CASE()函数。


SELECT employee_id,last_name,department_id,salary, CASE department_id 
	WHEN 10 THEN
		salary * 1.1
	WHEN 20 THEN
		salary * 1.2
	ELSE
		salary * 1.3
END "details"
FROM employees;

COUNT(*)

方式一：count（*）
方式二：count（1）
方式三：COUNT（具体字段）：不一定对


# 注意计算指定字段出现的次数是，是不包括NULL值的
SELECT COUNT(employee_id),COUNT(salary),COUNT(2 * salary),COUNT(1)
FROM employees;

SELECT commission_pct
FROM employees;

# AVG = SUM / COUNT。我认为VG(commission_pct),SUM(commission_pct)/COUNT(commission_pct)相等，因为SUM不会低级的直接用
# +来计算吧，而是忽略NULL，而COUNT也是忽略，那么应该相等。
SELECT AVG(salary),SUM(salary)/COUNT(salary),
AVG(commission_pct),SUM(commission_pct)/COUNT(commission_pct),
SUM(commission_pct) / 107
FROM employees;

# 需求：查询公司中平均奖金率，因为COUNT是不计算空值的
SELECT SUM(commission_pct) / COUNT(IFNULL(commission_pct,0)),
AVG(IFNULL(commission_pct,0))
FROM employees;

如果需要同济表中的记录数，使用COUNT（*），COUNT（1），COUNT（具体字段）哪个效率最高呢？

如果使用的时MyISAM存储引擎，则三者效率相同都是O(1)

如果使用的是InnoDB存储引擎，则三者效率：COUNT(*) = COUNT(1) > COUNT(字段)

GROUP BY的使用

用于可以使用GROUP BY子句将表中的数据分成若干组

结论：


# 结论1：SELECT中SELECT中出现的非组函数的字段必须声明在GROUp by中
# 反之，GROUp BY中声明的字段可以不出现在SELECT中

# 结论2：GROUP BY声明在FROM后面、WHRER后面，ORDER BY前面、LIMIT前面

# 结论3：ORDER BY中使用WITH ROLLUP,将所有的数据加起来然后平均


# 需求：查询各个部门的平均工资，最高工资
SELECT department_id,AVG(salary)
FROm employees
GROUP BY department_id;

# 需求查询哥哥department_id,job_id的平均工资
SELECT department_id,job_id,AVG(salary)
FROM employees
GROUP BY department_id,job_id;

SELECT department_id,job_id,AVG(salary)
FROM employees
GROUP BY job_id,department_id;

# 结论1：SELECT中SELECT中出现的非组函数的字段必须声明在GROUp by中
# 反之，GROUp BY中声明的字段可以不出现在SELECT中

# 结论2：GROUP BY声明在FROM后面、WHRER后面，ORDER BY前面、LIMIT前面

# 结论3：ORDER BY中使用WITH ROLLUP,将所有的数据加起来然后平均
SELECT department_id, AVG(salary)
FROm employees
GROUP BY department_id WITH ROLLUP;

注意 当使用ROLLUP时，不能同时使用ORDER BY子句进行结果排序，即ROLLUP和ORDER BY是互相排斥 的。

HAVING

作用用于过滤数据

结论：当过滤条件中有聚合函数时，则此过滤条件必须在HAVINg中。
当过滤条件中没有聚合函数时，则此过滤条件声明在WHERE中。HAVING都可以，但是必须使用WHERE


# 要求3：开发中，我们使用HAVIng的前提是SQL中使用了GROUP BY。

# 练习：查询部门id为10，20，30，40中的最高工资比10000的部门信息

# 方式1：

SELECT department_id,MAX(salary)
FROM employees
WHERE department_id in (10,20,30,40)
GROUP BY department_id
HAVING MAX(salary) > 10000;

# 方式2：推荐使用执行效率高

SELECT department_id,MAX(salary)
FROM employees
GROUP BY department_id
HAVING MAX(salary) > 10000 AND department_id in (10,20,30,40);

WHERE 与 HAVING的对比

区别1：WHERE 可以直接使用表中的字段作为筛选条件，但不能使用分组中的计算函数作为筛选条件； HAVING 必须要与 GROUP BY 配合使用，可以把分组计算的函数和分组字段作为筛选条件。

区别2：如果需要通过连接从关联表中获取需要的数据，WHERE 是先筛选后连接，而 HAVING 是先连接 后筛选。

SELECT语句的执行的过程

关键字的顺序：


SELECT ... FROM ... WHERE ... GROUP BY ... HAVING ... ORDER BY ... LIMIT...

SELECT的执行顺序


FROM ...., -> ON ->  (LEFT/RIGHT JOIN) -> WHERE -> GROUP BY -> HAVING -> SELECT  -> DISTINCT -> ORDER BY -> LIMIT

首先FROM提取表中数据，ON 对数据进行过滤，LEFT/RIGHT JOIN对过滤后的表进行连接，WHERE对已经连接的表进行进一步过滤。之后GROUP BY 对过滤的表进行分组，然后HAVING进行进一步过滤，接下来SELECT对虚拟表查询数据，DISTINCT对查询后的表的数据进行去重操作，之后ORDER BY对表进行排序，最后LIMIT选择展示的内容条目数量。

# 6.查询各个管理者手下员工的最低工资，其中最低工资不能低于6000，没有管理者的员工不计算在内

错误总结：
第7题时没有考虑全部这个问题，所以没有想到外连接。其次COUNT的使用也有注意点，即有些部门是没有人的，如果使用COUNT(*)他会将外连接之后形成的数据计算进去，导致统计的人数为1，从而错误。所以可以使用count(employee_id)来进行人数的统计从而忽略NULL的情况。

第八题与第7题一样没有考虑到外连接的问题。此外它的答案我不太认可，因为在我的理解中，job_id也应该作为分组标准之一。
