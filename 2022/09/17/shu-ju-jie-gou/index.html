数组存放结点的数据以及一个指向存储孩子位置的链表。链表存放该结点孩子的数组下标。

int Patition(A,low,high){
   while(low < high){
      int pivort = A[low];
      while(low < high && A[high] >= privort){
         high--;
      }
      A[low] = A[high];
      while(low < high && A[low] <= privort){
         low ++;
      }
      A[high] = A[low];
   }
   A[low] = pivort;
   return low;
}

void Merge(int *a, int low, int mid, int high){
   for(int i = low; i < high; i++){
      b[i] = a[i];
   }
   
   for(int i = low, int k = i, int j = mid + 1; i <= mid && j <= high;k++){
      if(b[i] >= b[j]){
         a[k] = b[i++];
      }else{
         a[k] = b[j++];
      }
   }
   while(i <= mid){
      a[k++] = b[i++];
   }
   while(j <= high){
      a[k++] = b[j++];
   }
}

void Merger(int *a, int low, int high){

8.6 几种排序算法的比较

本章的主要内容观看王道。

平均情况下归并排序的空间复杂度是O(n)O(n)O(n)，归并排序和快速排序的最差空间复杂度是O(N)O(N)O(N)

  1. 时间复杂度

简单选择排序与序列的初始状态无关。希尔排序作为插入排序的拓展，对大规模的排序都有很高的效率。

归并排序与快速排序一样基于分治的思想，但是归并排序由于分割子序列与初始序列的排序无关，因此最好最差和平局的时间复杂度都为O(nlog2n)O(nlog_2n)O(nlog2​n)

  1. 空间复杂度：

简单选择排序、 插入排序、冒泡排序、希尔排序和堆排序都需要接著常熟个辅助空间。快速排序在空间上只是用一个雄安的辅助站，用于实现递归。平局下大小为O(nlong2(n))O(nlong_2(n))O(nlong2​(n))，最坏情况下为O(n)。归并排序则需要辅助队列及您修改元素的赋值，。

  1. 稳定性

插入排序、冒泡排序、归并排序、基数排序是稳定的。

简单选择排序、快速排序、希尔排序和堆排序都是不稳定的。

平均时间复杂度O(nlon2n)O(nlon_2n)O(nlon2​n)的稳定排序算法只有归并排序。

  1. 过程上来说

冒泡排序和堆排序在每趟处理后都能产生当前最大值或最小值， 快速排序一趟处理就能确定一个元素的最终位置。

  1. 内部排序算法的应用：

（1）若n较小，可采用直接插入排序或简单选择排序。由于直接插入排序所需的记录移动次数叫简单选择网排序的的多，因而当记录本身信息量较大时，使用简单选择排序比较好。
（2）若文件的初始状态已按照关键字基本有序，则选用直接插入或者冒泡排序
